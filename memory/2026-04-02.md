## 🎯 **重大方法论突破：信息收集的"元思考"机制**

### 📝 **讨论背景**
晚上与啸讨论了信息收集的方法论，提出了建立"元思考"机制的重要性。

### 💡 **核心收获**

#### **1. 元思考流程（5个关键问题）**
啸提出的思考框架：
1. **这个任务属于哪个领域？**
2. **这个领域有哪些权威网站？**
3. **有哪些重要的组织机构，比如大公司等？**
4. **有什么相关的知名人物？**
5. **有什么标志性事件？**

**核心原则**：先思考再行动，多问问题，不怕问问题，就怕没问题，不要蛮干。

#### **2. 信息源的重新定义（关键洞察）**
啸指出：**信息源不一定是网站**，而是多维度的：
- 组织机构/公司的信息
- 人物的评论和观点
- 各种形式的内容载体

#### **3. 工作原则明确化**
啸明确了四个关键问题的处理原则：

**原则1：信息源验证**
- 通过网络搜索背景资料判断
- 无法判断时向啸汇报，等待调研结果

**原则2：检索停止条件（"够"的标准）**
- **权威覆盖**：找到3-5个高权威信息源（权威性≥4）
- **观点饱和**：新信息源不再提供新观点/数据
- **时间效率**：连续30分钟无实质性新发现
- **特殊情况**：搜索1小时后仍少于3个可靠信息源 → 立即汇报

**原则3：成本控制**
- 以免费替代品为主
- 付费信息源信息附在报告里，标注价值评估
- 未来有条件时考虑加入付费信息源

**原则4：伦理处理**
- 收集阶段：全面收集，不做过滤
- 应用阶段：根据用途进行伦理评估
- 标注提醒：在报告中标注潜在伦理问题

### 🛠️ **落实到工作流程**

#### **信息收集任务执行清单**

**阶段一：任务分析（5-10分钟）**
- [ ] 明确任务领域和边界
- [ ] 识别潜在信息源类型（机构、人物、媒体等）
- [ ] 制定分层搜索策略
- [ ] 预估时间投入（根据紧急程度）

**阶段二：信息收集（动态调整）**
- [ ] 第一轮：高权威信息源（目标：3-5个）
- [ ] 第二轮：补充信息源（目标：观点覆盖）
- [ ] 第三轮：验证和交叉核对
- [ ] 持续评估"够"的标准

**阶段三：整理汇报**
- [ ] 标注信息源权威性
- [ ] 标注付费/伦理提醒
- [ ] 总结收集成果和局限
- [ ] 提出后续建议（如需付费信息源）
