# 2026年4月3日 - 经验教训总结

## 🎯 今日核心教训

### 1. **路径错误：文件夹层级关系**
- **错误**：在`04_定稿/`中使用`03_配图/文件名.jpg`
- **正确**：使用`../03_配图/文件名.jpg`
- **教训**：必须理解相对路径，考虑文件夹层级关系
- **影响**：看似小错误，但影响图片显示，体现专业度

### 2. **API参数混淆：文生图 vs 图生图**
- **关键区别**：
  - 文生图：无`image`参数
  - 图生图：有`image`参数（单张字符串或多张数组）
- **错误**：应该用图生图却用了文生图
- **后果**：IP形象特征丢失
- **教训**：必须仔细检查API参数，理解不同模式的区别

### 3. **多张参考图用法**
- **正确格式**：`image: ["url1", "url2"]`（数组）
- **错误理解**：以为单张参考图能生成多个IP形象
- **教训**：多IP形象必须用多张参考图

### 4. **未优先使用skill**
- **问题**：自己写脚本而不是使用image-generate skill
- **风险**：容易出错，不符合规范
- **改进**：优先使用skill，确保参数正确

## 💡 重要技术经验

### 火山引擎API正确用法
```python
# 文生图（无IP形象）
{
    'prompt': '描述',
    # 没有image参数
}

# 图生图（单IP形象）
{
    'prompt': '描述',
    'image': '单张图片URL',
}

# 图生图（多IP形象）✅
{
    'prompt': '描述',
    'image': ["url1", "url2"],  # 数组！
}
```

### 图片尺寸规范（灵活应用）
- **正文配图**：1080×720px（严格）
- **结尾图**：1080px宽，高度灵活（特殊）
- **装饰图**：根据实际情况处理
- **规则不是死的，要理解背后的原因**
