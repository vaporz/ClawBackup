# 日记系统 - 三层记忆管理系统

> 创建时间：2026-03-29
> 更新者：Jarvis

## 系统概述

这是一个三层记忆管理系统，旨在解决日记记录中的记忆准确性问题。系统分为：

1. **工作日志** - 客观事实记录
2. **知识库** - 背景知识存储  
3. **个人日记** - 主观感受表达

## 目录结构

```
.diary/
├── 2026/03/                    # 个人日记（永久保留）
│   └── 2026-03-28.md
├── logs/                       # 工作日志（3个月保留）
│   └── 2026-03/
│       ├── 2026-03-28-agent-communication-fix.md
│       └── 2026-03-28-editorial-sop-established.md
├── knowledge/                  # 知识库（6个月保留）
│   ├── agent-communication.md
│   └── editorial-workflow.md
├── reviews/                    # 定期回顾
│   ├── 2026-03-周回顾.md
│   └── 2026-03-月回顾.md
└── cleanup.sh                  # 清理脚本
```

## 职责边界

### 1. 工作日志 (.diary/logs/)
- **目的**：为写日记提供事实依据
- **内容**：具体任务、决策、问题解决过程
- **保留期限**：3个月
- **示例**：`2026-03-28-agent-communication-fix.md`

### 2. 知识库 (.diary/knowledge/)
- **目的**：为写日记提供背景知识
- **内容**：规范、流程、最佳实践
- **保留期限**：6个月
- **示例**：`agent-communication.md`

### 3. 个人日记 (.diary/YYYY/MM/)
- **目的**：个人反思和情感记录
- **内容**：主观感受、思考过程、人生感悟
- **保留期限**：永久
- **示例**：`2026-03-28.md`

## 使用流程

### 日常使用
1. **完成任务时** → 记录到 `logs/`
2. **建立规范时** → 同步到 `knowledge/`
3. **写日记时** → 参考 `logs/` 和 `knowledge/` 中的事实
4. **定期回顾时** → 提炼人生原则

### 清理机制
- **每月1日凌晨3点**：自动执行 `cleanup.sh`
- **工作日志**：保留3个月，按月清理
- **知识库**：保留6个月，按月清理
- **个人日记**：永久保留

## 与MEMORY.md的关系

### MEMORY.md（长期团队记忆）
- **目的**：团队协作和关键决策记录
- **内容**：关键里程碑、重要决策、长期偏好
- **更新频率**：每周或重大事件后
- **保留期限**：永久（但需要定期整理）

### 分工明确
- **.diary/**：为**写日记**服务的临时存储
- **MEMORY.md**：为**团队协作**服务的长期记忆
- 两者职责分明，不会混淆

## 问题解决

### 常见问题
1. **日记记录不准确** → 检查 `logs/` 中的工作记录
2. **忘记背景知识** → 查阅 `knowledge/` 中的规范
3. **文件过多** → 等待每月1日自动清理

### 手动干预
如果需要立即清理：
```bash
# 清理超过3个月的日志
find .diary/logs -name "*-*" -type d -mtime +90 -exec rm -rf {} \;

# 清理超过6个月的知识库
find .diary/knowledge -name "*.md" -mtime +180 -delete
```
