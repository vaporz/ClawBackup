# 务实工作模式原则

> **创建时间**：2026年3月30日
> **创建背景**：OpenClaw底层通信技术尚不成熟，回归务实工作模式
> **核心理念**：各司其职，用户亲自协调

---

## 核心原则

### 1. 接受技术现状
- **现实**：OpenClaw底层通信尚不成熟
- **表现**：Gateway不稳定、会话管理不完善、状态同步延迟
- **对策**：不依赖复杂协作，回归简单直接的工作模式

### 2. 简化优于复杂
- **复杂模式**：agent间自动协作、多层通信、状态依赖
- **简单模式**：你指挥，agent执行，直接通信
- **选择**：技术不成熟时，简单方案更可靠

### 3. 聚焦实际结果
- **不纠结**：系统状态、通信错误、技术细节
- **关注**：任务是否完成、质量如何、用时多少
- **标准**：能完成工作就行，过程状态不重要

---

## 工作模式

### 角色定位
| 角色 | 职责 | 工作方式 |
|------|------|---------|
| **你（指挥官）** | 分配任务、检查质量、整合结果 | 直接指挥各agent |
| **agent（专家）** | 专注专业领域，做好本职工作 | 接收指令，完成任务，汇报结果 |
| **Jarvis（协调员）** | 总协调、技术问题、系统维护 | 支持你协调，处理技术问题 |

### 通信模式
```
理想：你 → agent A → agent B → agent C
现实：你 → agent A（完成A部分）
      你 → agent B（完成B部分）
      你 → agent C（完成C部分）
      你 → 整合最终结果
```

### 任务分配原则
1. **一次一事**：每个任务明确具体
2. **直接通信**：你→agent，避免中转
3. **明确要求**：任务描述清晰，交付标准明确
4. **合理预期**：根据agent能力设定合理目标

## 效率评估

### 当前状态
- **时间节省**：已节省80%时间
- **复杂度**：从10降到3（满分10）
- **可靠性**：从60%提升到95%+（直接通信）
- **控制权**：用户完全掌握协调权

### 优势
1. **简单可靠**：你发消息，agent必响应
2. **控制权最大化**：所有协调权在你手中
3. **认知负担低**：不纠结技术细节，专注任务
4. **灵活适应**：随时可调整，等技术成熟再升级

### 接受的不完美
1. **系统有点乱**：状态报告不准确，但能干活
2. **通信不完美**：agent间协作困难，但直接通信可靠
3. **需要你协调**：增加一些协调工作，但保证结果

## 特殊情况处理

### 如果agent无响应
1. **等待1-2分钟**
2. **重发一次消息**
3. **如仍无响应，联系Jarvis协调**
4. **考虑换其他方式完成**

### 如果任务复杂
1. **拆分成小任务**
2. **分步分配给agent**
3. **你负责整合中间结果**
4. **最终由你合成完整成果**

### 如果需要多agent协作
1. **你分别给每个agent分配具体部分**
2. **各自完成后汇报给你**
3. **你负责整合和协调**
4. **避免agent间直接复杂通信**
