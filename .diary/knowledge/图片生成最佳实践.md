# 图片生成最佳实践 - 基于火山引擎经验

## 📋 概述
本文档总结了使用火山引擎Seedream 5.0 Lite API生成信息图的最佳实践，特别针对中文内容和技术概念可视化。

## 🎯 核心问题与解决方案

### 问题1：中文小字渲染问题（鬼画符）
**现象**：稍小的中文文字在生成图片中变成无法识别的符号。

**根本原因**：
- 火山引擎对中文小字渲染能力有限
- 像素密度不足时问题更严重

**解决方案**：
1. **文字分层策略**：
   - 🏷️ **大字用中文**：标题、副标题等醒目文字
   - 🔤 **小字用英文**：标签、数据、说明文字
   - 📝 **混合使用**：保持中文主体，保证可读性

2. **比例优化**：
   - 📐 **2:3竖版**：相比1:1，同样像素下文字更大
   - 🖼️ **高分辨率**：使用2K或更高分辨率

3. **视觉替代**：
   - 🎨 **用图形代替文字**：箭头、图标、颜色编码
   - 📊 **用图表代替数据**：可视化代替文字描述
   - 😊 **用表情传达情感**：IP形象的表情动作

### 问题2：IP形象不准确
**现象**：生成的猫狗形象不是Flash和Neo。

**解决方案**：
1. **图生图API**：使用参考图URL
2. **多参考图**：同时提供Flash和Neo两张参考图
3. **详细描述**：在提示词中明确描述角色特征

### 问题3：概念表达不清晰
**现象**：技术概念在图片中表达不明确。

**解决方案**：
1. **对比构图**：左右/上下对比布局
2. **颜色编码**：红色=问题，绿色=解决方案
3. **流程箭头**：用箭头展示变化过程
4. **数据可视化**：用图表代替纯数字

## 📊 技术参数配置

### API调用参数
```python
payload = {
    'model': 'doubao-seedream-5-0-260128',
    'prompt': '详细提示词',
    'image': [FLASH_URL, NEO_URL],  # 多参考图
    'sequential_image_generation': 'disabled',
    'response_format': 'url',
    'size': '2K',  # 或 '1920x1920'
    'stream': False,
    'watermark': False
}
```

### 参考图URL
- Flash (狸花猫): `http://images.kanxingkeji.com/flash.jpg`
- Neo (边牧): `http://images.kanxingkeji.com/neo.jpg`

### 尺寸要求
- **最小像素**：3686400像素（约1920x1920）
- **推荐比例**：2:3竖版适合手机阅读
- **分辨率**：2K或更高保证清晰度

## 🐱🐶 IP形象情感设计指南

### 情感对比原则
通过Flash和Neo的表情对比，增强故事性：

| 主题 | Flash表情 | Neo表情 | 情感对比 |
|------|-----------|----------|----------|
| 网络优化 | 😫 overwhelmed | 😊 confident | 压力 vs 从容 |
| 对话效率 | 😖 frustrated | 😌 relaxed | 挫败 vs 轻松 |
| 监控透明 | 😵 confused | 👁️ focused | 困惑 vs 专注 |
| 协作规范 | 😰 stressed | 🎯 precise | 压力 vs 精准 |
| 实施策略 | 😱 terrified | 🧗 confident | 恐惧 vs 自信 |

### 动作设计建议
- **Flash**：通常代表问题方，动作表现困惑、压力、重复劳动
- **Neo**：通常代表解决方案，动作表现自信、指挥、高效

## 🖼️ 构图与视觉层次

### 推荐构图比例
```
顶部 15%：标题区域（大字中文）
中部 60%：概念对比图（核心内容）
底部 20%：数据面板（英文小字）
角落 5%：IP形象引导图标
```

### 视觉编码系统
- **颜色**：
  - 🔴 红色：问题、警告、风险
  - 🟢 绿色：解决方案、成功、安全
  - 🔵 蓝色：科技感、专业、主色调
  - 🟠 橙色：对比、强调、活力

- **图形**：
  - ⬇️ 向下箭头：问题、下降、风险
  - ⬆️ 向上箭头：解决方案、提升、改进
  - ❓ 问号：未知、困惑、问题
  - ✅ 对勾：解决、正确、完成

## 📈 性能优化建议

### 避免API限流
1. **请求间隔**：每张图之间等待8-10秒
2. **分批处理**：一次不要生成超过5张图
3. **错误重试**：实现简单的重试机制
4. **超时设置**：设置合理的超时时间（60-90秒）

### 提示词优化
1. **长度控制**：1000-1500字符为宜
2. **结构清晰**：分段明确，逻辑清晰
3. **关键词突出**：重要要求放在前面
4. **避免矛盾**：不同要求之间不要冲突
