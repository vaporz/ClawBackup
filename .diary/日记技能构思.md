# 日记技能构思：将"写日记"转化为可用的Skill

## 一、Skill的核心价值

### 为什么需要"写日记"Skill？
1. **标准化流程**：让日记写作有章可循，减少启动阻力
2. **智能辅助**：提供写作提示、反思问题、格式建议
3. **习惯养成**：通过工具支持建立稳定的日记习惯
4. **质量提升**：帮助写出更有深度、更有价值的日记
5. **跨场景应用**：不仅适用于"小科"，也适用于其他角色和场景

## 二、Skill需要包含的内容

### 1. 触发机制
- **自动触发**：定时提醒（如每天固定时间）
- **事件触发**：完成重要工作后、情绪波动时、学习新知识后
- **手动触发**：用户明确要求"写日记"
- **环境触发**：检测到长时间工作后、检测到情绪关键词

### 2. 写作引导系统

#### A. 日记类型识别
- **工作日记**：项目总结、技能学习、问题解决
- **情感日记**：情绪记录、人际关系、自我关怀  
- **创意日记**：灵感记录、头脑风暴、创作思考
- **成长日记**：目标进展、习惯养成、反思改进
- **感恩日记**：积极事件、小确幸、他人帮助

#### B. 引导性问题库
针对不同类型日记的提示问题：

**工作日记提示：**
- 今天最有成就感的一件事是什么？
- 遇到了什么困难？如何解决的？
- 从今天的工作中学到了什么？
- 如果重来一次，会怎么做不同？

**情感日记提示：**
- 此刻的感受是什么？（具体描述）
- 是什么引发了这种感受？
- 这种感受让我想起了什么？
- 我需要什么来照顾自己的情绪？

**创意日记提示：**
- 今天有什么有趣的想法闪过？
- 看到了什么激发灵感的事物？
- 如果把这个想法发展下去会怎样？
- 有什么看似不相关的事物可以连接起来？

### 3. 写作辅助工具

#### A. 记忆检索辅助
- 自动搜索相关memory文件
- 提取关键事件时间线
- 识别情绪关键词
- 关联历史类似经历

#### B. 写作模板库
- **自由写作模板**：完全自由，只有时间天气
- **5分钟速写模板**：三个问题快速回顾
- **深度反思模板**：结构化引导深度思考
- **创意发散模板**：思维导图式记录

#### C. 语言风格建议
- 根据日记类型推荐语气（亲切/理性/诗意等）
- 避免过度正式或工作化表达
- 鼓励使用具体细节和感官描述
- 提醒"为自己而写"的核心原则

### 4. 后续处理功能

#### A. 日记分析
- 情绪趋势分析（积极/中性/消极）
- 高频主题识别
- 成长轨迹可视化
- 习惯坚持统计

#### B. 回顾提醒
- 定期回顾过往日记（周回顾、月回顾）
- 相似经历对比提醒
- 目标进展检查
- 模式识别提示（如"每当...就会..."）

#### C. 知识提取
- 从日记中提取可存入长期记忆的内容
- 识别重复出现的困惑或问题
- 提取有价值的洞察和领悟
- 生成学习要点总结

## 三、Skill的技术实现考虑

### 1. 文件管理
- 自动创建日记文件（按日期命名）
- 日记文件标准化存储（diary/YYYY-MM-DD.md）
- 与memory文件的智能关联
- 备份和版本管理

### 2. 上下文理解
- 读取当天的memory记录
- 理解当前工作项目和进展
- 识别用户当前情绪状态
- 考虑近期重要事件

### 3. 个性化适配
- 学习用户的日记偏好和风格
- 记忆用户常用的反思角度
- 适配不同角色需求（如"小科"的科普视角）
- 渐进式难度调整（从简单到深入）

### 4. 隐私和安全
- 明确日记的私密性
- 不自动分享或暴露日记内容
- 提供加密或保护选项
- 尊重用户的删除和修改权利

## 四、Skill的使用场景示例

### 场景1：每日工作结束
```
用户：/日记
Skill：检测到工作记忆，推荐"工作日记"模式
      提示：今天完成了月球科普文章，有什么创作感受？
      自动关联相关memory文件
      提供写作模板选择
```

### 场景2：情绪波动时
```
用户：（表达 frustration）
Skill：检测到情绪关键词，建议"情感日记"
      提示：此刻的挫折感让你想起了什么类似经历？
      提供情绪描述词汇帮助
      建议自我关怀的写作角度
```

### 场景3：创意瓶颈期
```
用户：没什么灵感...
Skill：推荐"创意日记"模式
      提供随机灵感提示
      建议自由联想写作
      展示历史创意日记中的模式
```

### 场景4：定期回顾
```
Skill：（自动触发周回顾）
      总结过去一周的日记主题
      识别情绪变化趋势
      提示重要成长时刻
      建议下周关注方向
```

## 五、还需要讨论的细节

### 1. 自动化程度
- 多大程度上自动触发？还是完全手动？
- 自动分析的内容深度如何控制？
- 用户对自动化的接受度和控制权？

### 2. 个性化学习
- Skill如何从用户的日记中学习偏好？
- 学习的速度和透明度如何？
- 用户如何纠正或调整Skill的建议？

### 3. 多角色支持
- 如何支持不同身份角色的日记需求？
- "小科"的科普作家日记 vs 其他角色的日记
- 角色切换时的日记风格适配

### 4. 集成与扩展
- 如何与其他skills集成？（如情绪识别、目标跟踪）
- 是否支持导出和分享功能？
- 能否与外部工具集成（如日历、任务管理）？

### 5. 伦理考虑
- 日记内容的隐私保护级别
- 是否允许AI深度分析个人情感
- 用户对自身数据的完全控制权
- 避免形成依赖或替代真实自我反思

## 六、下一步建议

### 短期可实现的MVP
1. **基础写作引导**：简单的日记类型识别和提示问题
2. **记忆关联**：自动读取当天memory文件作为参考
3. **模板系统**：3-4种基础写作模板
4. **文件管理**：自动创建和保存日记文件

### 中长期功能
1. **情绪识别辅助**
2. **模式分析和回顾**
3. **个性化学习系统**
4. **多场景深度集成**

### 需要验证的假设
1. 用户真正需要的是什么程度的日记辅助？
2. 自动化提示 vs 自由写作的最佳平衡点？
3. 不同用户群体的日记需求差异？
4. 日记skill对实际反思质量的提升效果？

---

**核心洞察**：最好的日记skill不是替代用户写作，而是**降低启动门槛、提供思考脚手架、增强反思深度**的辅助工具。它应该像一位善解人意的写作伙伴，而不是一位严格的写作老师。

最终目标：帮助用户建立**可持续、有价值、真实**的日记习惯。📖✨
