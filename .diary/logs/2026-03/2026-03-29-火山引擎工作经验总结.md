## 🔍 火山引擎的"脾气"（能力边界）

### ✅ 能做什么（优势）
1. **基础构图**：左右/上下对比布局
2. **颜色表达**：红绿颜色编码（问题/解决）
3. **IP形象**：参考图生图，表情动作表达
4. **大字标题**：中文大字渲染尚可
5. **简单图形**：箭头、图标、基础图表

### ❌ 不能做什么（限制）
1. **小字中文**：鬼画符问题严重
2. **复杂布局**：细节容易丢失
3. **精确图表**：数据可视化精度有限
4. **专业信息图**：达不到商业级质量
5. **文字密集**：文字越多效果越差

### ⚠️ 需要注意（特性）
1. **像素要求**：至少3686400像素（约1920x1920）
2. **提示词长度**：1000-1500字符效果较好
3. **请求频率**：需要间隔8-10秒避免限流
4. **参考图质量**：需要公网可访问的清晰图片
5. **生成时间**：每张图约10-20秒

## 🎨 "迁就"模型的创作策略

### 策略一：文字极简化
**原则**：能用图形就不用文字，必须用文字就简化

**具体做法**：
1. **只保留大字标题**：中文，醒目，像素多
2. **去掉所有小字**：标签、数据、说明用英文或图形
3. **视觉替代文字**：
   - 颜色代替状态（红=问题，绿=解决）
   - 箭头代替流程（↓=下降，↑=提升）
   - 图标代替概念（❓=未知，✅=完成）
   - 表情代替情感（😫=压力，😊=自信）

### 策略二：视觉强化
**原则**：让图形自己说话

**具体做法**：
1. **强烈对比**：左右红绿对比，上下问题解决对比
2. **情感叙事**：用Flash和Neo的表情讲故事
3. **颜色编码**：建立一致的视觉语言系统
4. **流程可视化**：用箭头和连线展示变化

### 策略三：复杂度控制
**原则**：一张图只讲一个故事

**具体做法**：
1. **单概念聚焦**：网络优化就只讲网络，不混入其他
2. **元素精简**：主要视觉元素不超过5个
3. **层次清晰**：标题→核心图→辅助信息
4. **布局简单**：避免复杂嵌套和重叠

### 策略四：IP情感化
**原则**：用角色情感增强故事性

**具体做法**：
1. **情感对比**：Flash（问题方）vs Neo（解决方）
2. **表情设计**：根据主题设计匹配的表情
3. **动作叙事**：通过动作展示问题和解决
4. **位置安排**：问题在左/上，解决在右/下

## 📝 提示词编写经验

### 有效结构
```
【主题描述】- 简单明确
【视觉要求】- 具体可执行
【IP设计】- 情感和动作
【文字策略】- 大字中文，小字英文
【风格规格】- 颜色、比例、布局
```

### 关键技巧
1. **明确指令**：用"Create a conceptual infographic about..."开头
2. **具体描述**：不要"好看的信息图"，要"左右对比，红色蜘蛛网vs绿色星型网"
3. **情感细节**：不要"Flash在左边"，要"Flash😫被红色混乱网络包围，表情压力"
4. **技术规格**：明确"2:3 vertical aspect ratio", "tech blue main color"
5. **避免矛盾**：不同要求之间要一致

### 长度控制
- **理想长度**：1000-1500字符
- **太短**：细节不足，AI自由发挥过多
- **太长**：重点模糊，可能被截断或忽略
