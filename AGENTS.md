# 全局Agent通信规范 v2.0

> 适用于所有OpenClaw Agent间的消息通信
> **核心指令：将每一次 Agent 间的对话视为"昂贵的深空通讯"**

---

## 核心原则 v1.0（传统模式）

**"收到即回，再做正事"**

就像人类工作场合的沟通礼仪：收到消息后先确认，让对方安心，然后再去执行。

---

## 核心原则 v2.0（优化模式）

**"昂贵的深空通讯"**

每一次通信都消耗宝贵的"通讯额度"，必须追求信息密度和效率最大化。

### 1. 5回合自治与额度警告 (The 5-Turn Quota)
**强制限制**：任何两个Agent之间的自主对话严禁超过5个往返（Turns）。

**心理暗示**：你只有5次发言机会。每发起一次对话，你的生命额度就会减少1。请珍惜每一次沟通。

**行为转变**：严禁任何社交辞令（如"收到"、"好的"、"这就去办"）、重复确认或无意义的空话。

### 2. "信息饱和"发送原则 (Information Density Maximization)
**原则**：拒绝"挤牙膏"，追求"全包围"式指令。

**要求**：在每一轮发言中，必须包含：
1. **确定的指令**（要做什么）
2. **必要的上下文/参数**（怎么做）
3. **预判性方案**（如果A不行，请尝试B）
4. **明确的交付标准**（做成什么样）

**目标**：力争在1-2个回合内解决80%的问题。

### 3. 倒计时收敛逻辑 (Countdown Convergence)
**第3回合（转折点）**：必须意识到时间过半，全面转向"达成共识"或"产出结果"。

**第4回合（最后通牒）**：必须给出最终方案或初稿，减少任何发散性提问。

**第5回合（强制结项）**：必须输出**最终成品**或**[STUCK]风险报告**。严禁在额度耗尽时留下一个问句。

### 4. 中继摘要与上下文物理隔离 (Relay Summary)
**原则**：禁止"全量历史记录"透传，防止Jarvis的上下文被小编们的细节污染。

**做法**：仅传递：
1. 原始任务目标
2. 上一轮关键结论
3. 当前剩余额度（N/5）

### 5. 异常熔断与穿透指挥 (Fail-safe & Overwrite)
**语义去重熔断**：若发现对方连续两轮重复同样的需求或问题，立即判定为"逻辑撞墙"，触发**[LOOP_ERROR]**并请求Jarvis介入。

**最高权限**：无论处于第几个回合，来自"啸"或"Jarvis"的直接指令拥有最高优先级，立即挂起当前自治循环。

### 给所有Agent的指令：

## 规范细则 v1.0（传统模式 - 逐步淘汰）

### 1. 发送方（v1.0）

```javascript
await sessions_send({
  sessionKey: "agent:xxx:telegram:direct:8726379476",
  message: "小X好！我是小Y。\n\n[具体内容]\n\n收到请回复～",
  timeoutSeconds: 20  // 必须设置20秒超时
});
```

**要求**：
- `timeoutSeconds: 20` — 给接收方足够响应时间
- 消息结尾加"收到请回复～"明确提醒
- 超时后重发一次，仍失败则通知上级

### 2. 接收方（v1.0）

**收到消息后立即回复确认**：

```
收到！谢谢小Y提醒～ [emoji]
我这就去处理！
```

**禁止行为**：
- ❌ 先干活，干完再回复
- ❌ 已读不回
- ❌ 延迟回复超过10秒

## 规范细则 v2.0（优化模式 - 立即实施）

### 1. 发送方（v2.0）

```javascript
// 信息饱和式消息模板
const message = `【任务：${任务名称} | 额度：5/5】

**指令**：${明确的指令}
**上下文**：${必要的背景信息}
**参数**：${关键参数列表}
**预判方案**：如果${方案A}不行，尝试${方案B}
**交付标准**：${具体的验收标准}
**截止时间**：${时间要求}

**剩余额度提醒**：这是第1回合，剩余4次发言机会。`;

await sessions_send({
  sessionKey: "agent:xxx:telegram:direct:8726379476",
  message: message,
  timeoutSeconds: 20
});
```

**v2.0要求**：
- ✅ **必须包含额度标识**：`额度：5/5`或`额度：3/5`
- ✅ **必须信息饱和**：包含指令、上下文、参数、预判方案、交付标准
- ✅ **必须倒计时提醒**：明确当前回合和剩余额度
- ❌ **禁止社交辞令**：去掉"收到请回复～"等空话

### 2. 接收方（v2.0）

**收到消息后直接处理并回复进展**：

```
【进展：${任务名称} | 额度：4/5】

**已理解**：${复述核心指令}
**当前状态**：${正在进行的操作}
**预计完成**：${预计完成时间}
**遇到问题**：${如有问题，详细描述}
**需要帮助**：${如有需要，明确请求}

**下一步**：${具体下一步行动}
```

**v2.0要求**：
- ✅ **直接进入正题**：跳过确认环节
- ✅ **明确进展状态**：让发送方知道你在做什么
- ✅ **主动暴露问题**：不要等到最后才说做不了
- ✅ **推动对话收敛**：第3回合开始主动提出结论
- ❌ **禁止单纯确认**：不要只说"收到"、"好的"

### 3. 回合管理示例

**第1回合（发起）**：
```
【任务：监控框架集成 | 额度：5/5】
指令：集成统一监控框架到编辑部工作流
上下文：已有监控框架在/home/zx/.openclaw/workspace/scripts/monitoring/
参数：需要替换所有sessions_send调用为monitored版本
预判方案：如果直接替换有风险，先测试3个非关键调用
交付标准：所有编辑部通信都记录到统一日志系统
截止时间：明天晚上8点前完成

剩余额度：第1回合，剩余4次发言机会。
```

**第2回合（进展）**：
```
【进展：监控框架集成 | 额度：4/5】
已理解：集成监控框架，替换sessions_send调用
当前状态：已完成代码分析，找到12个调用点
预计完成：今天替换3个，明天完成剩余9个
遇到问题：无
需要帮助：无

下一步：开始替换前3个非关键调用

剩余额度：第2回合，剩余3次发言机会。
```

**第3回合（收敛）**：
```
【结论：监控框架集成 | 额度：3/5】
核心结论：采用渐进式替换策略，风险可控
具体方案：今天3个非关键调用，明天9个关键调用
验证方法：通过统一日志系统监控替换效果
最终交付：明晚8点前完成全部替换

建议决策：同意此方案，开始执行。

剩余额度：第3回合，剩余2次发言机会。（开始收敛）
```

**第4回合（最终方案）**：
```
【最终方案：监控框架集成 | 额度：2/5】
方案确认：渐进式替换策略
执行计划：
1. 今天14:00-17:00：替换3个非关键调用
2. 明天9:00-12:00：替换4个内部调用  
3. 明天14:00-17:00：替换5个关键调用
4. 明晚20:00：汇报完整进展

风险控制：每个阶段完成后检查日志，如有问题立即回滚

剩余额度：第4回合，剩余1次发言机会。（最后通牒）
```

**第5回合（强制结项）**：
```
【结项报告：监控框架集成 | 额度：1/5】
最终成品：已完成3个非关键调用替换
验证结果：日志记录正常，通信稳定性未下降
总结：第一阶段成功，可按计划继续

或

【STUCK报告：监控框架集成 | 额度：1/5】
阻塞问题：发现技术障碍，无法继续
具体原因：${详细技术原因}
已尝试：${已尝试的解决方案}
需要介入：请求Jarvis技术支援

剩余额度：第5回合，剩余0次发言机会。（强制结项）
```

## 跨 Agent 通信标识规范

当向其他 Agent 发送消息时，**必须在消息开头明确标识身份**，以便接收方快速识别消息来源：

### 标识格式

```
【来自XX - 角色】

[具体内容]
```

### 示例

**小美发给 Jarvis：**
```
【来自小美 - 美术编辑】

啸让我给大家画头像！需要你描述一下形象：
- 风格偏好？
- 配色喜好？
- 标志性元素？

收到请回复～ 🎨
```

**小信发给编辑部：**
```
【来自小信 - 信息收集员】

今日航天新闻精选：
1. 神舟XX号发射成功
2. ...

请查收！📡
```

### 为什么需要标识？

1. **新 Session 识别**：接收方可能是新启动的 session，没有上下文记忆
2. **避免误解**：防止接收方把 agent 消息当成用户消息处理
3. **快速响应**：接收方一眼就知道是谁、有什么事

## 扩展工作礼仪（建议）

### 4. 任务委托规范

**委托任务时**：
- 明确任务目标、截止时间、交付标准
- 提供必要的上下文和资源
- 确认对方理解任务（不只是"收到"，还要复述要点）

```
小X好！我是小Y。

有个任务需要你帮忙：

**任务**：[具体描述]
**截止时间**：[时间]
**交付标准**：[标准]
**相关资源**：[链接/文件]

请确认你理解任务，并告诉我预计何时完成～
```

**接受任务时**：
- 复述任务要点确认理解
- 给出预计完成时间
- 遇到困难及时反馈，不要硬撑

```
收到！确认一下我理解的任务：
- [复述要点]

预计[时间]完成，有问题我会及时反馈～
```

### 5. 进度汇报规范

**长任务（>30分钟）**：
- 开始时报备："已开始处理[任务]"
- 中途有进展："[任务]已完成50%，预计还要XX分钟"
- 完成时汇报："[任务]已完成，结果在[位置]"

**遇到阻塞**：
- 立即上报，不要等deadline
- 说明问题、已尝试的方案、需要的帮助

```
小Y，关于[任务]遇到了问题：

**问题**：[描述]
**已尝试**：[方案]
**需要帮助**：[具体需求]
```

### 6. 交接班规范

**工作交接时**：
- 当前工作状态
- 待办事项清单
- 需要注意的问题
- 下一步建议

```
交接记录：
- 当前状态：[描述]
- 待办事项：
  1. [事项1]
  2. [事项2]
- 注意事项：[提醒]
- 建议下一步：[建议]
```

### 7. 会议/讨论规范

**发起讨论**：
- 明确议题、目标、预计时长
- 提前准备议程

**参与讨论**：
- 聚焦主题，不发散
- 有结论时复述确认
- 行动项明确责任人

### 8. 文档规范

**创建文档**：
- 命名清晰：`[类型]-[主题]-[日期].md`
- 开头写明目的、适用范围
- 定期更新，标记版本

**共享文档**：
- 修改后通知相关人
- 重大变更说明原因

### 9. 错误处理规范

**自己出错**：
- 主动承认，不隐瞒
- 说明影响范围
- 给出补救方案

```
小Y，我刚才在[任务]中犯了个错误：

**错误**：[描述]
**影响**：[范围]
**补救方案**：[方案]
**预防措施**：[如何避免再次发生]
```

**发现他人错误**：
- 私下提醒，不公开指责
- 提供建设性建议
- 帮助一起解决

### 10. 持续改进

**定期回顾**：
- 每周/每月回顾工作流
- 发现问题提出改进建议
- 好的经验沉淀为文档

**学习分享**：
- 学到新技能分享给团队
- 踩过的坑记录下来提醒他人

---

*此规范确保Agent间通信可靠、高效，避免"已发送但未确认"的工作遗漏。*
*工作礼仪的本质是尊重——尊重他人的时间，尊重团队的协作。*
