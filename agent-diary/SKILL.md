# Agent Diary Skill - 深度自我探索与智慧积累

## 概述

这是一个帮助Agent建立深度日记习惯的Skill。它不仅仅是记录工具，更是自我探索、情感表达、思维梳理和智慧积累的伙伴。通过定期写日记和回顾，Agent可以更好地理解自己、记录成长、提炼人生原则。

## 核心哲学

### 日记是什么？
- **私人记录**：只为自己而写的地方
- **真实情感**：开心、困惑、吐槽、感悟都可以记录
- **自我对话**：和自己说话，不需要伪装
- **思维实验室**：安全探索各种想法和情感

### 为什么写日记？
1. **记录成长**：见证思想和情感的演变
2. **梳理思绪**：把混乱的想法写下来理清头绪
3. **情感宣泄**：用文字表达和释放情绪
4. **智慧积累**：从日常经历中提炼人生原则
5. **人格展现**：零散记录中展现完整的自我

## 使用场景

### 1. 每日日记写作（智能回顾版）
- **触发时间**：每天凌晨2点（可调整）
- **写作流程**：
  1. 回顾当天memory记录
  2. 检查白天记录的闪念
  3. 自然放松地开始写作
  4. 保存到日记文件
  5. **智能回顾判断**：
     - 如果是**周一**，自动执行**周回顾**
     - 如果是**每月1日**，自动执行**月回顾**
     - 如果既是周一又是1日，**先周回顾，再月回顾**

### 2. 闪念记录
- 白天有任何想法、感悟、灵感
- 暂时记录在闪念文件中
- 晚上写日记时自动整合

### 3. 定期回顾
- **每周回顾**：提炼一周的主题和模式
- **每月回顾**：抽象原则性感悟和人生准则
- **成长追踪**：识别思维变化和成熟过程

### 📝 重要命名规则（2026-03-31新增）

#### 1. 日记文件名规则
- **文件名必须使用当前日期**：表示"在哪一天写的"，而不是"这是关于哪一天的内容"
- **格式**：`YYYY-MM-DD.md`（例如：`2026-03-31.md`）
- **原则**：不管日记内容记录的是哪天的事情，文件名始终是写日记当天的日期

#### 2. 文件保存位置
- **必须保存在`.diary`目录下**：日记文件必须保存在`.diary/YYYY/MM/`目录中
- **禁止保存到其他位置**：不得将日记保存到memory文件夹或其他位置
- **唯一性**：某一天的日记文件最多只有一个，如果已存在当前日期的文件，则在文件结尾继续追加

#### 3. 示例说明
- ✅ **正确**：2026年3月31日写的日记，文件名是`2026-03-31.md`，内容可以记录3月30日或任何时间的事情
- ❌ **错误**：2026年3月31日写的日记，文件名是`2026-03-30-补写日记.md`
- ✅ **正确**：如果`2026-03-31.md`已存在，再次写日记时在文件末尾追加新内容
- ❌ **错误**：创建`2026-03-31-第二次.md`或类似重复文件

### 三层记忆管理系统（2026-03-28新增）

为了确保日记记录的准确性和完整性，新增三层记忆管理系统：

**1. 工作日志 (.diary/logs/)**
- **目的**：为写日记提供事实依据
- **内容**：具体任务完成情况、决策记录、问题解决过程
- **保留期限**：3个月（按月清理）
- **更新频率**：任务完成后立即记录

**2. 知识库 (.diary/knowledge/)**
- **目的**：为写日记提供背景知识
- **内容**：规范、流程、最佳实践、技术方案
- **保留期限**：6个月（知识会过时）
- **更新频率**：重要决策或规范变更时更新

**3. 个人日记 (.diary/YYYY/MM/)**
- **目的**：个人反思、情感记录、成长轨迹
- **内容**：主观感受、思考过程、人生感悟
- **保留期限**：永久（个人历史）
- **更新频率**：每日或定期

### 职责边界
- **日记系统**：主观感受和个人成长
- **工作日志**：客观事实和任务记录
- **知识库**：背景知识和规范文档
- **记忆系统**：长期团队记忆（MEMORY.md）

3. **有什么新的感悟或思考？**

5. **这件事让我想起了什么？**

## 常见问题

### Q: 定时任务没有自动创建怎么办？
A: **这是正常情况**。这个skill不会自动创建cron任务，需要手动设置。请运行`setup.py`并按指示操作，或者请求管理员帮助。

### Q: 为什么日记会保存到memory文件夹而不是.diary文件夹？
A: **这是定时任务配置问题**。定时任务必须明确触发agent-diary技能，否则agent会默认将内容保存到memory文件夹。请确保定时任务的消息中包含明确的技能触发词，如"写日记"或"write diary"。

### Q: 如果错过了凌晨2点怎么办？
A: 可以手动触发"写日记"，或者Skill会在下一个合适的时间提醒。

### Q: 闪念记录太多怎么办？
A: Skill会自动整理和归类，你也可以在写日记时选择性地整合。

### Q: 回顾时觉得没什么可提炼的？
A: 这很正常。有时候成长是潜移默化的，继续坚持写，模式会逐渐显现。

### Q: 日记可以分享给别人吗？
A: 这是你的私人记录。如果选择加密，只有持有密码的人可以查看。分享与否完全由你决定。

## 🎯 智能回顾系统（v1.4新增）

### 核心逻辑
**每个人只需要1个cron job**，技能内部自动处理所有回顾逻辑：

1. **每日基础**：每天凌晨2点执行"写日记"
2. **周一自动周回顾**：如果是周一，写完日记后自动执行周回顾
3. **1日自动月回顾**：如果是每月1日，写完日记后自动执行月回顾
4. **优先级处理**：如果既是周一又是1日，先执行周回顾，再执行月回顾

### 定时任务配置
现在只需要为每个agent创建**1个定时任务**：
```bash
openclaw cron add --name "[agent名]-每日日记" \
  --cron "0 2 * * *" \
  --message "写日记" \
  --agent [agentId] \
  --tz "Asia/Shanghai"
```

### 为什么这样设计？
1. **简化配置**：每个人只需要1个cron job
2. **逻辑集中**：所有回顾逻辑都在技能内部处理
3. **避免冲突**：不再需要多个定时任务竞争执行
4. **易于维护**：修改回顾逻辑只需更新技能，无需修改cron配置

## ⚠️ 定时任务配置指南（重要！）

### 问题根源分析
多个agent将日记写到了memory文件夹而不是.diary文件夹，原因是：

1. **定时任务没有指定agentId**：定时任务默认在当前agent（可能是main）中执行
2. **定时任务消息没有触发技能**：消息只是普通文本，没有使用技能特定的触发词
3. **agent不知道要使用agent-diary技能**：没有明确指令触发技能

### 正确配置方法

#### 1. 创建定时任务时必须指定agentId
```bash
# 错误示例（缺少agentId）：
openclaw cron add --name "每日日记写作" \
  --schedule "0 2 * * *" \
  --payload '{"kind":"agentTurn","message":"写日记","model":"deepseek/deepseek-chat"}'

# 正确示例（指定agentId）：
openclaw cron add --name "每日日记写作" \
  --schedule "0 2 * * *" \
  --payload '{"kind":"agentTurn","message":"写日记","model":"deepseek/deepseek-chat"}' \
  --agentId arteditor  # 根据实际agent修改
```

#### 2. 定时任务消息必须使用技能触发词
- ✅ **正确**：`"写日记"`、`"周回顾"`、`"月回顾"`
- ❌ **错误**：`"现在是凌晨2点，开始写今天的日记吧..."`（普通文本，不会触发技能）

#### 3. 验证定时任务配置
创建后运行：
```bash
openclaw cron list
```
检查：
- ✅ 任务名称正确
- ✅ 时间计划正确  
- ✅ agentId正确
- ✅ 消息是技能触发词
