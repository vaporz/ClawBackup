# HEARTBEAT.md - 个人助理与系统运行状态监控

## 🎯 监控范围
**仅限**: 个人助理相关 + 系统运行状态
**排除**: 编辑部内部工作细节

## 📋 监控任务清单

### 🔄 每30分钟检查（系统健康）
1. **OpenClaw系统状态**
   - Gateway服务运行状态
   - 各agent基础存活状态
   - 系统资源使用情况（CPU/内存）

2. **通信通道状态**
   - Telegram主账号连接状态
   - Feishu连接状态
   - 基础通信功能验证

3. **存储与备份**
   - 工作区磁盘空间监控
   - 备份任务执行状态验证
   - 关键文件完整性检查

4. **安全与更新**
   - 安全警告状态监控
   - 可用更新检查
   - 配置有效性验证

### 📅 每日定时检查

#### 早晨检查（08:00）
1. **系统启动检查**
   - 验证夜间备份是否成功
   - 检查各服务自动启动状态
   - 准备当日系统运行报告

2. **个人日程提醒**
   - 当日重要会议提醒
   - 截止任务提醒
   - 习惯养成提醒（如喝水）

#### 午间检查（12:00）
1. **系统性能检查**
   - 上午运行性能分析
   - 错误日志快速扫描
   - 资源使用趋势分析

2. **个人效率提醒**
   - 休息提醒
   - 任务进度简要检查
   - 下午工作准备提醒

#### 晚间检查（20:00）
1. **系统维护检查**
   - 当日运行总结
   - 临时文件清理建议
   - 次日维护计划

2. **个人总结提醒**
   - 当日任务完成情况
   - 明日计划提醒
   - 系统使用反馈收集

## 🚨 告警阈值

### 立即通知的严重问题
1. **系统服务宕机** - Gateway或核心服务停止
2. **存储危机** - 磁盘使用超过95%
3. **安全漏洞** - 检测到严重安全风险
4. **备份失败** - 连续备份任务失败

### 需要关注的警告问题
1. **性能下降** - 响应时间明显变慢
2. **资源紧张** - 内存/CPU持续高使用率
3. **通道不稳定** - 通信通道间歇性问题
4. **配置异常** - 检测到异常配置

## 📊 报告格式

```
[时间] 系统状态检查报告
🏥 系统健康: [状态]
📊 资源使用: CPU [%] / 内存 [%] / 磁盘 [%]
📡 通信状态: Telegram [状态] / Feishu [状态]
📁 存储状态: [可用空间] / [告警状态]
🔄 任务状态: Cron [正常/异常] / 备份 [状态]

⚠️ 需要注意:
1. [问题1]
2. [问题2]

💡 建议行动:
1. [建议1]
2. [建议2]
```
